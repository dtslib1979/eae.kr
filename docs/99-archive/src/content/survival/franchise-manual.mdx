## In Simple Terms

A franchise works because the manual works. McDonald's can open in any country because the operations manual specifies everything — from fryer temperature to greeting scripts.

CLAUDE.md is that manual for a content factory. Every repository has one. It tells the AI agent exactly how to operate: what is allowed, what is forbidden, what checks to run, what format to follow.

23 repositories share the same constitutional framework. Fork them to a new account, and the AI agent reads the CLAUDE.md and operates identically. No training needed. No onboarding. The manual IS the franchise.
</Part1>

## System Architecture

<Accordion>
  <AccordionItem title="1. Constitutional Framework">
  Every CLAUDE.md contains three articles:

  **Article 0: Editorial Technique** — the meta philosophy
  "Deconstruct originals, recombine into new meaning"

  **Article 1: Repository is a Novel** — narrative rules
  Commits are sentences, branches are chapters, git log is the plot

  **Article 2: Matrix Architecture** — operational rules
  ERP axis (ledger) + FAB axis (manufacturing) + 4 principles
  </AccordionItem>

  <AccordionItem title="2. Auto-Check System">
  CLAUDE.md is not documentation — it is enforcement:

  | Trigger | Check | Violation Response |
  |---------|-------|--------------------|
  | Before commit | Reason in message? | "Missing evidence" warning |
  | reset --hard request | Can revert instead? | Block, suggest revert |
  | New file added | BOM updated? | "BOM not updated" warning |
  | Session start | WIP branches count | Cleanup recommendation if 3+ |
  | Cross-repo work | Via script/manifest? | "Interface bypass" warning |

  The AI agent performs these checks automatically. No human discipline required.
  </AccordionItem>

  <AccordionItem title="3. Portability Test">
  What happens when you fork all 28 repos to a new account:

  1. CLAUDE.md files transfer with the fork (they are just files)
  2. AI agent reads them on first session
  3. Same rules apply immediately
  4. GitHub Actions CI runs on the new account
  5. Sync scripts need only path variable updates

  Total setup time for a new operator: token configuration + remote URLs. The operating logic transfers automatically because it lives in the files, not in anyone's memory.
  </AccordionItem>

  <AccordionItem title="4. Per-Repo Specialization">
  While the constitutional framework is shared, each repo adds specialized rules:

  - eae.kr: Publisher agent protocol, design system locks, English-only, Part1/2/3
  - parksy-audio: Musician deployment pipeline, MIDI rendering rules
  - dtslib-apk-lab: APK build protocol, 3-Check 2-Spot 1-Commit
  - dtslib-cloud-appstore: Browser=Studio framing, PWA constraints

  The franchise manual has a common core (constitution) plus local supplements (repo-specific rules). Same pattern as franchise operations worldwide.
  </AccordionItem>

## Why AI Makes This Possible

Before AI agents, a franchise manual was only useful if the new operator could read and follow it. Human compliance is unreliable — people skip steps, forget rules, develop habits that diverge from the manual.

AI agents do not skip steps. They read CLAUDE.md at the start of every session and enforce every rule, every time. The franchise manual becomes self-enforcing infrastructure.

This is the survival technique insight: the combination of CLAUDE.md (franchise manual) + AI agent (perfect compliance) + GitHub (portable infrastructure) creates a content factory that can be replicated to any operator with near-zero training cost.

The 10-20 year horizon becomes realistic because the system does not depend on one person's discipline. It depends on files that enforce themselves through AI.

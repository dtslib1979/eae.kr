## In Simple Terms

You pay for video editing software. You pay for audio processing. You pay for image manipulation. You pay for transcription services.

Or you open Chrome.

Chrome has 14 built-in engines: Speech-to-Text, Text-to-Speech, Canvas, WebGL, Camera, Picture-in-Picture, Web Audio, Bluetooth, WebAssembly, WebCodecs, MIDI, Gemini Nano, MediaRecorder, and File System Access.

Each one replaces a paid tool. Combined, they form a complete broadcast production studio — for free. The browser is not a viewer. The browser is a studio. You just need to know which engines to combine.
</Part1>

  <AccordionItem title="2. Technical Constraints (By Design)">
  The constraint list is a feature, not a limitation:

  - Zero webpack, vite, or npm
  - Zero TypeScript
  - Zero SPA frameworks
  - Zero login or accounts
  - FFmpeg WASM locked to 0.11.0 (no 0.12.x)
  - Pure HTML/JS/CSS + CDN only
  - Each tool is independent (no shared state)

  Why? Because these constraints ensure every tool works on any device, loads instantly, and requires zero setup. A broadcast production tool that needs npm install is not a production tool — it is a development project.
  </AccordionItem>

  <AccordionItem title="3. Termux = CLI Factory, Browser = Real-time Studio">
  The role division between Termux and browser is deliberate:

  - **Termux (CLI)**: Batch processing, automation, deployment, git operations
  - **Browser (Studio)**: Real-time editing, preview, interactive production

  They complement, not compete. Termux handles the factory floor (mass production). Browser handles the studio (creative decisions in real-time).
  </AccordionItem>

  <AccordionItem title="4. Design Filter">
  Every tool proposal passes one filter:

  "Does this produce broadcast material?"

  If yes: build it.
  If no: reject it.

  This filter has rejected dozens of ideas that were technically interesting but produced nothing usable for broadcast. The browser is a studio, not a playground.
  </AccordionItem>

## The Economics of Free

14 browser engines cost nothing. CDN hosting costs nothing. GitHub Pages costs nothing. The entire broadcast production infrastructure runs on free tiers.

This is not frugality — it is architectural philosophy. Paid tools create vendor lock-in. Free browser APIs are standardized, portable, and permanent. A tool built on Web Audio API will work in any browser, on any device, for as long as browsers exist.

Operational technique means choosing infrastructure that does not depend on anyone's pricing decisions. The browser is the most stable, most portable, most cost-effective studio ever built. You just have to learn to use the engines it already contains.

# Part 1 – Grandpa Mode

- Patchtech은 “개발자”가 아니라 “AI 관리자”가 되는 방식이다.  
- 모바일에서 말하면 AI가 설계하고, PC가 생산한다.  
- PWA와 APK를 목적에 따라 분리해 생산하는 체제다.  
- 비용은 0에 수렴하고, 속도는 점점 가속된다.  
- 결국 ‘개인 소프트웨어 공장’이 된다.

# Part 2 – System Architect Mode

## 1. Problem Statement
모바일은 입력엔 강하지만 컴파일이 안 된다.  
클라우드는 편하지만 비용이 쌓인다.  
앱 마켓은 개인화 워크플로우 배포엔 비효율적이다.

Patchtech은 이 세 문제를 구조적으로 재배치한다.

## 2. Patchtech Architecture (R&R)

| Layer | Component | Device | Function |
|-------|-----------|--------|----------|
| Command | User (CEO) | Mobile | 발상·지시·결정 |
| Strategy | ChatGPT | Cloud | 요구분석·백서화 |
| Architecture | Gemini | Cloud | 기술 검증·지시서 최적화 |
| Development | Claude MCP | Local PC | 코드 작성·파일 제어 |
| Build & CI | Local PC | Local | 빌드·APK 패키징·PWA 배포 |
| Repository | GitHub | Cloud | 버전 관리 및 중앙 허브 |

핵심 원리:  
**모바일이 명령하고 → AI가 구조화하고 → PC가 생산한다.**

## 3. Dual Track System

### A) PWA Track
- 목적: 대시보드, 유틸리티, 텍스트 처리  
- 배포: Vercel  
- 장점: 설치 불필요, 즉시 업데이트  

### B) APK Track
- 목적: 카메라·S Pen·FFmpeg 같은 고성능 작업  
- 빌드: Flutter + Local PC  
- 배포: USB sideload 또는 Drive  

## 4. Zero-Cost Build System
- 클라우드 CI 대신 집 PC를 Build Node로 사용  
- Claude MCP가 로컬 파일을 직접 조작  
- 사용자는 모바일에서 지시만 내려도 생산이 자동 진행  
- 비용: **$0 (전기세 제외)**  

## 5. Value Proposition
- **소프트웨어 주권 확보**  
- **워크플로우의 자산화**  
- **기획→설계→생산의 시간 단축**  
- **AI-Native 개인 개발 공장 완성**
